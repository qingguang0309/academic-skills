# 设计系统与中文排版规则

slidekit.js 已把这里的大多数规则做成机制;本文档解释"为什么",以及少数需要作者自己判断的地方。

## 画布与网格

- 16:9 宽屏,13.33 × 7.5 in(`LAYOUT_WIDE`)。国内会议室与线上会议的绝对主流。
- 左右边距 0.72 in,内容区宽 12.0 in;内容区底界 6.88 in,页脚基线 7.06 in。
- 块间距 0.26 in;双栏列间距 0.45 in。所有页面同一网格——观众感知不到网格,但感知得到"每页都很稳"。

## 字阶(pt,13.33 in 画布)

| 角色 | 字号 | 说明 |
|---|---|---|
| 封面主标题 | 36 | 粗体,白字,最多三行 |
| 页标题(action title) | 25 | 粗体,主色,至多两行,超两行 slidekit 会警告 |
| 章节页大号数字 | 96 | 极浅主题色,当背景图形用 |
| 章节标题 | 30 | 粗体 |
| 正文 | 17 | 溢出时布局引擎最多降到 ×0.85 |
| 辅文/卡片正文 | 12.5–14 | |
| 图注/表注 | 11.5 | 灰色,居中 |
| 页脚 | 9.5 | 极浅灰,存在但不打扰 |

层级只靠字号 + 字重 + 颜色,不靠换字体。全 deck 一套中文字体 + 一套西文字体。

## 主题与用色纪律

三套内置主题(THEMES):`azure` 藏青·金(通用)、`pine` 墨绿·赭(材料/能源)、`plum` 绛紫·杏(生医/化学)。每套 = 深主色 + 中亮强调色 + 一个暖色点缀 + 灰阶。

- **深主色**只用于:封面/结束页底色、页标题、表格粗线、章节标题。
- **强调色**只用于:数字编号、大数字卡数值、章节导航当前项、callout 标签——即"要观众先看"的地方。
- **暖色**只用于小方块母题与个别 chip,面积永远很小。这个 0.1 in 的小方块是全 deck 的视觉签名,出现在封面、目录、章节页、卡片标题、bullets 行首——不要换成别的形状,不要放大。
- 白底内容页 + 深色封面/结束页的"三明治"结构。内容页禁止大面积色块和满版图片。
- 单位有 VI 主色时,在 THEMES 里加一套(照抄字段结构),不要临时改散落的颜色。

## 中文排版规则(slidekit 机制 + 作者责任)

**机制已保证的**:

- 汉字、全角标点、中文引号破折号 → 中文字体(默认 Microsoft YaHei);拉丁、数字、半角符号 → 西文字体(默认 Arial)。混排文本自动分 run,不需要也不允许手动指定 fontFace。
- 表格用三线表:顶线、底线粗(主色),栏目线细,行间浅线,无竖线——与学位论文/学报规范一致。
- 页码 "03 / 16"、页脚短题、章节 breadcrumb 自动生成。

**作者要自己把住的**:

- 全角标点:中文句子里用 ,。;:、""(),不要混半角逗号句号。数字与单位之间的空格用在西文段里(如 "5% EDS"),slidekit 会给它西文字体。
- 标题断行:两行标题在语义边界断,用 `\n` 手动控制;不要让"的"、"与"落在行首行尾。
- 行长:正文一行不超过 ~28 个汉字当量;更长就该分栏或分页。
- 字体回退:Microsoft YaHei 在装了 Office 的 Windows/Mac 都有;若用户指定 PingFang SC/思源黑体,提醒跨平台观感差异。LibreOffice 渲染预览用的是替身字体,字间距以 PowerPoint 实际打开为准。

## 英文模式(lang:'en')

同一套版式与网格,差异只在:固定文案换英文(Contents/References/Thank You)、中西文都走 latin 字体、目录字距不加宽。英文 action title 用句子式大小写(Sentence case),不要 Title Case。

## 内容纪律(与版式同等重要)

- 每分钟至多一页内容页;15 min 会议 talk ≈ 12–14 页内容页。
- 一页一个论点;一页至多一个 exhibit(图/表/大数字组三选一为主角)。
- 图上要有"看这里"的标注(paper-figures 生成时就该带上);借用的图页内标出处,文末参考文献页给全条目。
- 结论页(2–4 条要点)放在参考文献之前,Q&A 时停留;结束页只做礼仪功能。
- 全篇未做实验的数值必须标"目标/预期",用 callout(tone:'warn') 声明。
